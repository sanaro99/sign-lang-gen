# Datasets

Which corpora this project can use for the text→gloss step, what each is honestly good for, and
how to build a split from it. Pairs with `ATTRIBUTION.md` (licences/terms) and
`docs/decision_log.md` (why).

**One rule above all:** nothing under `data/` is ever committed (`.gitignore` anchors `/data/`).
Sign-language corpora carry research-only or unstated terms; keep every corpus local.

---

## In plain words

To teach and grade an English→ASL-gloss model we need pairs of *(English sentence, gloss)*. The
catch is where the gloss comes from. Some corpora have gloss written by **Deaf signers** (real, but
small and access-gated); one big corpus has gloss **generated by a rule from English** (large and
instant, but not real ASL). A high score on the synthetic one does not prove the system produces real
ASL. So we use the synthetic corpus to build and test the plumbing, and real-gloss corpora — even
small ones — to check whether the pipeline works on genuine ASL notation.

---

## The corpora

### 1. ASLG-PC12 — synthetic, large, instant *(primary bring-up)*
- ~87.7k English–gloss pairs, `achrafothman/aslg_pc12` on Hugging Face; `load_aslg_pc12()`.
- **Gloss is rule-generated** from POS-tagged English (`X-I`, `DESC-`, English word order, kept
  punctuation) — *not* written by Deaf signers. High BLEU here does **not** demonstrate real ASL
  quality (Yin et al. 2021; Desai et al. 2024). Prompt: `prompts/baseline_gloss.md`.
- Split: `python scripts/download_data.py` → `data/processed/{example_pool,test}.jsonl`.

### 2. NCSLGR — real ASL, small, usable now *(real-ASL testbed)*
- ~1,875 English–gloss pairs from Boston University (Neidle et al.), **the ASLLRP family**. Gloss is
  **Deaf-produced**, in true ASLLRP conventions (`fs-TONY`, `IX-1p`, `POSS-1p`, `#ALL`, `(2h)`,
  depicting constructions `DCL:`/`BCL:`/`ICL:` and `5"…"`) with real ASL grammar. `load_ncslgr()`.
- **How we get it without the gate:** the ELAN annotation export (`main gloss` + `English
  translation` tiers) is publicly mirrored by DePaul (`asl.cs.depaul.edu/corpus/elanBUcorpus.zip`,
  ~5 MB, **no video**). The HF `ncslgr/ncslgr` script path no longer works (`datasets` ≥4 dropped
  loading scripts), so `load_ncslgr()` downloads + parses the EAF files directly (stdlib +
  `defusedxml`), caching to gitignored `data/raw/ncslgr/`.
- **Relationship to what we're waiting for — be explicit.** NCSLGR is **not** an independent
  dataset: it is the same Boston University / Neidle / ASLLRP project as the gated ASLLRP Sign Bank
  (its XML also lives on `secrets.rutgers.edu/dai`). It does not substitute for the Sign Bank
  dictionary the reproduction needs, and it may overlap the paper's own data. Treat it as a *preview
  of real ASL gloss*, not as "the reproduction on other data."
- **Licence:** the BU download page states no explicit terms (contact Prof. Neidle). We therefore
  treat NCSLGR **exactly like ASLLRP** — local research/education use only, gitignored, never
  committed, cited to Neidle/Vogler et al.
- **Do NOT score it with `baseline_gloss.md`.** That prompt targets ASLG-PC12 pseudo-gloss; against
  real ASLLRP notation BLEU collapses on *notation alone*, not quality — the exact trap
  `docs/decision_log.md` records for ASLG-PC12. Use the real-ASL prompt `prompts/asllrp_gloss.md`
  (PROVISIONAL) and retrieve real NCSLGR pairs so the notation is taught in-context
  (`configs/ncslgr_rag.yaml`). Any BLEU from it is **preliminary** (provisional prompt, small n, not
  the paper's ASLLRP split).
- **Split:** `python scripts/download_data.py --dataset ncslgr` → `data/processed/ncslgr/…`
  (document-level: whole stories held out as test; ~1,432 pool / ~249 test). Rebuild the index with
  `make index` pointed at the NCSLGR pool before an RAG run.
- **Signing-NMM tiers:** the EAF files also carry `yes-no question`, `wh question`,
  `conditional/when`, and `negative` tiers (`load_ncslgr(include_nmm=True)`). These annotate the
  **signing**, which the paper's Step 4 corrected *away from* toward the English text — so they are
  **not** valid gold for our text-derived NMM labels (they fire on embedded questions our text rubric
  scores 0). Useful only as a *comparison* (how often does signing-NMM match text-NMM?), never as the
  gold source in `scripts/evaluate.py`.

### 3. ASLLRP DAI 2 corpus — the paper's real source *(reproduction target — access granted)*
- The paper's actual Module-1 data: real, Deaf-produced ASL from the ASLLRP SignStream 3 Corpus,
  obtained through the Data Access Interface (https://dai.cs.rutgers.edu/dai/s/dai). `load_asllrp()`.
- **Format:** each downloaded *collection* is a `.zip` holding an `xml_extract_*.xml` (annotations),
  a `.ssc` file, and a nested video zip — we read **only the XML**. Each `<UTTERANCE>` = one sentence:
  `<TRANSLATION>` (English) + an ordered run of `<SIGN><LABEL>` gloss tokens (kept in document order —
  already chronological), each tagged with `SIGN_TYPE` (Lexical / Fingerspelled / Classifier / Loan /
  Gesture / Number). Sentence-type `<NON_MANUAL>` tiers (`wh question`, `negative`, `conditional/when`,
  `yes-no question`) map to our four NMM labels via `load_asllrp(include_nmm=True)` — **signing-based,
  so a starting point/comparison, not final gold** (the paper's Step 4 re-labels to the English text).
- **Status:** 4 collections downloaded (`data/dai/*.zip`, gitignored) → **156 pairs**. The full corpus
  is 84 SignStream files (paper: ~2,119 pairs before cleaning) — export the rest the same way.
- **Split:** `python scripts/download_data.py --dataset asllrp` → `data/processed/asllrp/` (document-level).
- **Licence:** research/education, **no redistribution** ([dai-terms](http://www.bu.edu/asllrp/dai-terms.html)) →
  local only, gitignored, never committed. This is the Step-1 extraction; the faithful reproduction also
  needs the Step-2 cleaning and the **Sign Bank dictionary** (a *separate* download — see
  `docs/faithful_reproduction_plan.md` Phase A).

### 4. RWTH-PHOENIX-Weather 2014T — real DGS gloss *(deferred option)*
- ~8.3k pairs (7,096 / 519 / 642), German Sign Language, real Deaf-transcribed gloss, weather domain;
  the standard SLT benchmark. Text annotations are available for research (video is 39 GB and not
  needed).
- **Deferred, not built.** It is **German→DGS**, so our English→ASL prompt and the four NMM labels do
  not apply — it would only test that the plumbing runs on another language, duplicating the
  exploratory ISL story with a weaker fit. Revisit only if we want a scoreable cross-lingual
  transfer demonstration.

### 5. Bangla text→gloss (arXiv 2504.02293, 2025) — *related work, not a data source*
- A recent LLM RAG few-shot text→gloss dataset (~4k **synthetic** glosses). Relevant to how we frame
  **contribution 1** (RAG for text→gloss is now established, not novel — see
  `docs/primary_source_findings.md` §3), not as a corpus we ingest.

## Adding another dataset

1. Write a `load_<name>()` in `src/aslgloss/data/loaders.py` returning `list[Example]`
   (`text`, `gloss`, `source`, optional `doc_id`/`sent_idx`), with a unit test that parses a tiny
   fixture (no network) — see `tests/test_ncslgr.py`.
2. Register a splitter in `scripts/download_data.py` (`_SPLITTERS`); it writes to
   `data/processed/<dataset>/` by default so it never clobbers the primary split.
3. If the gloss conventions differ, add a matching prompt in `prompts/` and a `configs/<name>.yaml`;
   do not reuse a prompt written for a different notation.
4. Record the licence/terms in `ATTRIBUTION.md` and the decision in `docs/decision_log.md`. If terms
   are unstated or research-only, it stays gitignored — like everything under `data/`.
